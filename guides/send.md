# How to remove your course from Send (send.now / send.cm)

> Canonical version: [https://delister.co/remove/send-now](https://delister.co/remove/send-now). This repo mirrors it for reference and machine-readability.

Send, which runs on send.now and send.cm, is a file host rather than a listing site. Piracy aggregators upload a course to Send and then link to it. Reaching Send directly removes the file itself, which is what ends the distribution.

## Does Send honor DMCA notices?

Yes. Send publishes a copyright contact and removes infringing files on a valid notice. Because Send holds the file itself, a notice here does more than a notice to the listing page ever could: it removes the thing people are downloading. We have filed against Send-hosted course files ourselves, in July 2026, and the route below is the one that worked.

## The steps

### 1. Find the exact Send file URL

Follow the download button on the listing until you land on a send.now or send.cm address. Copy the full URL. A single course is often split across many part files on Send, so collect every one; a notice that lists them all removes the whole set together.

### 2. Send a DMCA notice to copyright@send.now

Email `copyright@send.now` and include:

- Your full legal name and country as the rights holder.
- Your course name and a one-line description of the work.
- Your official sales page URL as the authorized example of the work.
- The exact infringing Send file URLs. One notice can carry the full set of part files.
- The two sworn statements a DMCA notice requires: a good-faith belief that the use is not authorized, and that the information is accurate and you are the rights holder or authorized to act.

Send it from an address on your own domain where a reply can reach you.

### 3. Confirm the removal

Re-open the file URLs after Send acts. A removed file returns a not-found or removed page instead of the download. Confirm at the file URL, not the listing page.

## Time cost

Around 20 minutes for a first notice, most of it spent collecting the part-file URLs behind the listing. Then a recheck a few days later, and again whenever a reupload appears.
